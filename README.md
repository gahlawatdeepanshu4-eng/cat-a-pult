# Cat-a-pult

Pull back the slingshot and fling a rock at the cats and T-rexes in the arena.
Twenty levels, and they get better at dodging every one.

A single-player landscape game for Android, installable to the home screen.

## Play locally

```bash
npm run serve
```

Then open http://localhost:5173 in a landscape phone-sized window.

To play on your phone, find your PC's local IP and open `http://<that-ip>:5173`
on the same wifi.

## Test

```bash
npm test
```

No dependencies. Tests run on Node's built-in runner.

## Regenerate the icons

```bash
npm run icons
```

The icons are generated by a raw PNG encoder built on Node's `zlib`, so there
is no binary asset that cannot be rebuilt from source.

## Rules

Pull back and let go, like a slingshot. **Drag direction aims, drag length
sets power, and there is no timer** — take as long as you like. A drag too
short to be a shot cancels instead of firing a dud.

Cat 20 points, T-rex 50. Clear every creature before the rocks run out.
Running out just restarts the level: no lives, no game over.

Twenty levels. Dodging ramps up from the start, jumping begins at level 2, and
from level 4 there are two flying cats and two flying T-rexes every level.

## How it fits together

- `src/aim.js` — drag vector to heading, elevation and power. Pure. The ghost
  arc and the real shot both call it, so the preview cannot lie.
- `src/ballistics.js` — pure 3D projectile maths. No canvas, no DOM.
- `src/project.js` — the perspective projection. The only module that knows how
  world space becomes screen space.
- `src/creatures.js` — cats and T-rexes: wander, jump, fly, dodge, hit tests.
- `src/levels.js` — the 20-level curve, as a formula rather than 20 blobs.
- `src/game.js` — the rules. Returns new state, never mutates.
- `src/render.js` — all drawing. Reads state, never writes it.
- `src/main.js` — the loop and screen wiring.

World space is y-up, with the sand at y=0 and z running into the screen.

### The arena is a wedge, not a box

A rock's horizontal reach at depth z is exactly `tan(MAX_HEADING) * z`, so the
aim cone is narrow close up and wide far away. Creatures must spawn, wander and
dodge inside that cone, or they end up visible but impossible to hit at any
aim. `xLimitAt(z)` in `creatures.js` enforces this, and the reachability tests
in `game.test.js` prove every creature on all 20 levels can actually be killed.

### Two rules worth keeping

- **Dodging is capped below certainty** and limited to one attempt per creature
  per shot. A target that always dodges could never be killed.
- **The service worker is network-first.** Cache-first pins players to whatever
  build they first loaded, so fixes never arrive however often they reload.

Tuning lives in `src/constants.js`.
